# Товарный Флоу (Order Flow) Спецификация

## Обзор
Эта спецификация описывает все возможные флоу заказа товара в приложении Kalashyulya от просмотра товара до получения готового заказа.

## Архитектурные компоненты
- **Frontend:** Nuxt 3 приложение
- **State Management:** Nuxt composables, ref, reactive
- **Database:** Firebase Realtime Database
- **Payment:** ЮKassa API (оплата картой)
- **Backend:** Яндекс Cloud Functions (webhook обработка)

## Основные участники флоу

### Пользователь
- Просматривает товары в магазине
- Выбирает способ заказа (прямая покупка или корзина)
- Оформляет заказ (заполняет форму)
- Олачивает заказ через ЮKassa
- Отслеживает статус заказа

### Продавец
- Получает уведомления о новых заказах через Telegram webhook
- Связывается с покупателем для уточнения деталей доставки
- Обновляет статус заказа в Firebase

### Системы
- Firebase Realtime Database: хранение заказов
- Яндекс Cloud Functions: обработка webhook от ЮKassa
- ЮKassa: обработка платежей, отправка webhook

## Флоу 1: Прямая покупка (Исторический)

### Описание
Пользователь нажимает кнопку "Купить" на карточке товара и сразу переходит к оплате.

### Точки входа
- Страница товара в магазине (`/shop`)
- Карточка товара (Item.vue)

### Процесс
1. **Выбор товара:** Пользователь просматривает товары, выбирает нужный
2. **Нажатие "Купить":** Пользователь нажимает кнопку "Купить" на карточке товара
3. **Открытие формы заказа:** Открывается форма заказа с предзаполненными данными
4. **Заполнение формы:** Пользователь заполняет контактные данные и способ доставки
5. **Создание заказа:** Заказ создается в Firebase
6. **Выбор способа оплаты:**
   - Онлайн оплата через ЮKassa
   - Оплата переводом вручную
7. **Оплата (если выбрано онлайн):** Редирект на страницу оплаты ЮKassa
8. **Успешная оплата:** Отображение страницы успешной оплаты с деталями заказа

### Компоненты
- **Item.vue** - карточка товара с кнопкой "Купить"
- **OrderForm.vue** - форма заказа (удалена в текущей реализации)
- **PaymentMethodSelector.vue** - выбор способа оплаты (удален)
- **PaymentSuccess.vue** - страница успешной оплаты
- **Links.vue (футер)** - соц. сети, реквизиты

### Выходы
- Страница успешной оплаты с деталями
- Переход в магазин или на отслеживание

### Данные
- Заказ создается в Firebase с полями: customer, purchase, status, totalPrice
- Payment ID сохраняется в Firebase после успешной оплаты

### Статусы заказа
- Новый заказ → В обработке → Оплачен
- Отменен → Отменен (если платеж не прошел)

## Флоу 2: Добавление в корзину (Текущий)

### Описание
Пользователь нажимает кнопку "Добавить в корзину" на карточке товара. Товар добавляется в корзину.

### Точки входа
- Страница товара в магазине (`/shop`)
- Карточка товара (Item.vue)

### Процесс
1. **Просмотр товаров:** Пользователь просматривает товары в магазине
2. **Выбор товара:** Пользователь выбирает интересующий товар
3. **Нажатие "Добавить в корзину":** Пользователь нажимает кнопку "Добавить в корзину"
4. **Добавление в корзину:** Товар добавляется в состояние корзины
5. **Отображение корзины:** Количество товаров в корзине обновляется в UI
6. **Переход в корзину:** Пользователь может перейти в корзину для оформления заказа

### Компоненты
- **Item.vue** - карточка товара с кнопками "Купить" и "Добавить в корзину"
- **Basket.vue** - страница корзины
- **BasketWidget.vue** - виджет корзины (header)
- **ShopList.vue** - список товаров

### Выходы
- Пользователь может оформить заказ из корзины
- Страница корзины показывает все добавленные товары

### Данные
- Корзина хранится в состоянии приложения (Nuxt composable)
- При оформлении заказа товары из корзины переходят в заказ

## Флоу 3: Оформление заказа из корзины (Текущий)

### Описание
Пользователь переходит в корзину и оформляет заказ, выбирая способ оплаты.

### Точки входа
- Страница корзины (`/basket`)
- Форма оформления заказа

### Процесс
1. **Просмотр корзины:** Пользователь переходит на страницу корзины
2. **Заполнение формы:** Пользователь заполняет контактные данные и способ доставки
3. **Выбор способа оплаты:**
   - Онлайн оплата через ЮKassa
   - Оплата переводом вручную
4. **Создание заказа:** Заказ создается в Firebase
5. **Оплата (если выбрано онлайн):**
   - Вызов YCF createPayment
   - Редирект на страницу оплаты ЮKassa
   - Возврат confirmationUrl (payment ID для виджета)
6. **Оплата через виджет:** Пользователь оплачивает в виджете ЮKassa
7. **Webhook после оплаты:** ЮKassa вызывает YCF processPayment
8. **Обновление статуса:** Webhook обновляет статус заказа на "Оплачен", сохраняет paymentId
9. **Успешная оплата:** Отображение страницы успешной оплаты с деталями заказа

### Компоненты
- **Basket.vue** - страница корзины с товарами
- **OrderForm.vue** - форма оформления заказа
- **PaymentMethodSelector.vue** - выбор способа оплаты (удален)
- **PaymentSuccess.vue** - страница успешной оплаты
- **AnimatedBlob.vue** - компонент успеха

### Выходы
- Страница успешной оплаты с деталями заказа
- Переход в магазин или на отслеживание

### Данные
- Заказ создается в Firebase с полями: customer, purchase, status, totalPrice, paymentId
- Payment ID сохраняется в Firebase после успешной оплаты через webhook

### Статусы заказа
- Новый заказ → Оплачен (онлайн оплата)
- В обработке → Оплачен
- Отменен → Отменен (если платеж не прошел или вручную)
- Новый заказ → В обработке → Оплачен (ручная оплата)

## Флоу 4: Успешная оплата и страница успеха

### Описание
После успешной оплаты через ЮKassa пользователь попадает на страницу успешной оплаты с деталями заказа.

### Точки входа
- Редирект с ЮKassa на `/shop/success?paymentId=...`
- Страница успеха (`/shop/success`)
- Страница успешной оплаты (`/shop/payment-success`)

### Процесс
1. **Оплата через виджет:** Пользователь оплачивает в виджете ЮKassa
2. **Webhook от ЮKassa:** ЮKassa вызывает webhook на YCF processPayment
3. **Обработка webhook:** YCF processPayment обрабатывает событие payment.succeeded
4. **Сохранение paymentId:** Webhook сохраняет paymentId в заказе в Firebase
5. **Обновление статуса:** Статус заказа обновляется на "Оплачен"
6. **Редирект на успех:** ЮKassa редиректит на `/shop/success?paymentId=...`
7. **Редирект на payment-success:** Страница `/shop/success` редиректит на `/shop/payment-success?paymentId=...`
8. **Отображение деталей:** Страница успешной оплаты загружает заказ из Firebase по paymentId и отображает детали

### Компоненты
- **PaymentSuccess.vue** - страница успешной оплаты
- **OrderTracking.vue** - страница отслеживания
- **Links.vue** - футер с ссылкой на отслеживание

### Выходы
- Пользователь видит детали заказа
- Номер для отслеживания (paymentId) сохранен для дальнейшего использования
- Контакты для связи с продавцом

## Флоу 5: Отслеживание заказа

### Описание
Пользователь может отслеживать статус заказа используя ID платежа.

### Точки входа
- Страница магазина (футер с ссылкой на отслеживание)
- Страница отслеживания (`/shop/tracking`)
- Firebase Database

### Процесс
1. **Переход на отслеживание:** Пользователь нажимает ссылку "Отслеживание заказа" в футере
2. **Ввод ID платежа:** Пользователь вводит ID платежа в форму поиска
3. **Поиск заказа:** Форма отправляет запрос в YCF processPayment (webhook уже обновил статус)
4. **Отображение результатов:** Компонент отображает:
   - Детали заказа (номер, товар, цена, статус, дата, доставка)
   - Информация о клиенте
5. **Обработка ошибок:** Если заказ не найден или ошибка Firebase - отображается сообщение об ошибке
6. **Повторный поиск:** Кнопка "Попробовать снова" при ошибке

### Компоненты
- **OrderTracking.vue** - страница отслеживания заказа
- **Links.vue** - футер с ссылкой на отслеживание

### Выходы
- Пользователь видит актуальный статус заказа
- Может повторно искать другие заказы

### Данные
- Payment ID используется как уникальный идентификатор для поиска заказа в Firebase

## Статусы заказа

| Статус | Описание | Цвет |
|---------|-----------|-------|
| Новый заказ | Заказ создан, ожидает оплаты | #6b7280 (синий) |
| В обработке | Оплата в процессе | #f59e0b (оранжевый) |
| Оплачен | Успешная оплата | #10b981 (зеленый) |
| Отменен | Отмена или неудачная оплата | #ef4444 (красный) |

## Интеграция с внешними системами

### ЮKassa
- **API:** Yookassa API v3
- **Способы оплаты:** Карта (виджет), СБП
- **Webhook:** POST запрос на настроенный URL при изменении статуса платежа
- **IDempotence Key:** Обязательно для создания платежей

### Яндекс Cloud Functions
- **createPayment:** Создание платежа
  - Вход: orderId, amount, description, returnUrl, userId, customer
  - Выход: confirmationUrl, paymentId, status
  - IDempotence Key: orderId + timestamp

- **processPayment:** Обработка webhook
  - Вход: event type, payment object
  - Дейсвия:
    - payment.succeeded → статус="Оплачен", сохранить paymentId
    - payment.canceled → статус="Отменен"
    - payment.waiting_for_capture → статус="В обработке"

## Безопасность

### Защита данных
- Firebase правила безопасности ограничивают доступ к данным
- Payment ID - безопасен как идентификатор для отслеживания (уникальный, 36 символов)
- Данные заказа не содержат конфиденциальной информации о кредитных картах

### Валидация
- Валидация полей заказа перед созданием
- Проверка наличия обязательных полей (email, phone)
- Валидация ID платежа перед поиском

### Webhook безопасность
- Проверка подписи webhook (опционально)
- Использование HmacSHA256 для валидации webhook от ЮKassa

- `YOOKASSA_SECRET_KEY` - Secret Key ЮKassa (в YCF)

- `FIREBASE_PRIVATE_KEY` - Приватный ключ Firebase (в YCF)
- `FIREBASE_CLIENT_EMAIL` - Email сервиса Firebase
- `FIREBASE_DATABASE_URL` - URL базы данных Firebase
