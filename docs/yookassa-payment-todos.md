## 🚨 КРИТИЧЕСКИЕ (High Priority)

### 1. Исправление несоответствия типа платежа и виджета
- **Статус:** ❌ Критическое несоответствие
- **Файлы:** `cloud-functions/create-payment/index.js`, `composables/useYookassaPayment.ts`
- **Описание:** Cloud function создает платеж с type: 'redirect', но виджет ожидает confirmation_token
- **Проблема:**
  - Cloud function возвращает `confirmation.confirmation_url` (для редиректа)
  - Виджет YookassaCheckout с `embedded: true` требует `confirmation_token` (для встроенного виджета)
  - По документации ЮKassa:
    - `type: 'redirect'` → возвращает `confirmation_url`
    - `type: 'embedded'` → возвращает `confirmation_token`
- **Решение:** Выбрать ОДИН подход:
  - **Вариант A (рекомендуется):** Использовать redirect вместо виджета
    - Изменить cloud function: вернуть `confirmationUrl: paymentData.confirmation.confirmation_url`
    - Удалить виджет и использовать `window.location.href = confirmationUrl`
    - ЮKassa редиректит на returnUrl после оплаты
    - Проще в реализации и отладке
  - **Вариант B:** Использовать встроенный виджет
    - Изменить cloud function: `confirmation: { type: 'embedded', return_url: returnUrl }`
    - Обновить виджет: использовать `confirmation_token` из ответа API
    - Сохранить токен для возможности восстановления виджета
- **Задачи:**
  - [ ] Принять решение о подходе (A или B)
  - [ ] Обновить `cloud-functions/create-payment/index.js`
  - [ ] Обновить `composables/useYookassaPayment.ts`
  - [ ] Обновить `components/shop/PaymentWidget.vue` (если используется)
  - [ ] Обновить `pages/shop/payment.vue` для правильной обработки
  - [ ] Протестировать полный флоу оплаты
