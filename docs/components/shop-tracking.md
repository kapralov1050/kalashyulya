# OrderTracking Component

## Описание
Компонент страницы отслеживания заказа, которая позволяет искать заказы по ID платежа.

## Путь
`pages/shop/tracking.vue`

## Props
Нет (автономный компонент страницы)

## Состояния (State)
- **searchPaymentId** - введенный ID платежа пользователем
- **loading** - загрузка данных заказа из Firebase
- **error** - ошибка при получении данных
- **order** - данные заказа загружены

## События (Events)
Нет (нет событий для родительских компонентов)

## Функционал
Компонент предоставляет форму поиска заказа по ID платежа и отображает детальную информацию о найденном заказе.

### Основные функции:
1. **Ввод ID платежа:**
   - Текстовое поле с плейсхолдером
   - Пример: "314c92fb-000f-5000-b000-1c4140d12338"
   - Валидация на пустоту перед поиском

2. **Поиск заказа:**
   - Кнопка "Найти заказ" (disabled при пустом вводе или загрузке)
   - Поиск в Firebase по `paymentId` (как в PaymentSuccess)
   - Валидация результатов поиска

3. **Отображение деталей заказа:**
   - Номер заказа (order.id)
   - Товар (order.purchase.order[0].title)
   - Цена (order.totalPrice)
   - Статус с цветной индикацией:
     * Оплачен - зеленый (#10b981)
     * В обработке - оранжевый (#f59e0b)
     * Отменен - красный (#ef4444)
     * Новый заказ - синий (#6b7280)
   - Дата заказа (formatDate)
   - Способ доставки (getDeliveryMethod)

4. **Отображение информации о клиенте:**
   - Имя (order.customer.name)
   - Email (order.customer.email)
   - Телефон (order.customer.phone || 'Не указан')
   - Способ связи (order.customer.userMessenger || 'Не указан')

5. **Обработка ошибок:**
   - Пустой ввод: "Введите ID платежа для поиска заказа."
   - Заказ не найден: "Заказ с таким ID не найден. Проверьте введенный ID или свяжитесь с поддержкой."
   - Ошибка Firebase: "Не удалось выполнить поиск. Попробуйте позже."

6. **Кнопка повторного поиска:**
   - "Попробовать снова" при ошибке
   - Очищает форму и состояние ошибки

7. **Хелпер функции:**
   - `formatOrderDate` - форматирование даты заказа
   - `getDeliveryMethod` - определение способа доставки
   - `getStatusClass` - получение CSS класса для статуса

## Данные (Data)
Компонент работает с той же структурой данных в Firebase, что и PaymentSuccess.

## Зависимости (Dependencies)
- Нет (автономный компонент)

## Интеграция с Firebase
Компонент использует Firebase Realtime Database для поиска заказов.

### Query Pattern
```javascript
db.ref('orders').orderByChild('paymentId').equalTo(searchPaymentId).once('value')
```

## UI/UX
- Форма поиска по центру страницы с белой карточкой на сером фоне
- Карточка заказа разделена на "Детали заказа" и "Информация о клиенте"
- Цветовая индикация статусов заказа
- Адаптивность для мобильных устройств (max-width: 800px)
- Поддержка темной темы (dark mode)
- Спиннер загрузки при поиске
- Кнопки действий с hover эффектами

## Особенности реализации
- **Безопасность:** Валидация входных данных, безопасное отображение данных из Firebase
- **Производительность:** Запрос к Firebase только при нажатии на кнопку "Найти заказ"
- **Доступность:** Читаемый и понятный интерфейс на русском языке
- **Обработка ошибок:** Комплексная система обработки ошибок с дружественными сообщениями

## Цветовая схема статусов
```css
.status-paid { color: #10b981; }      // Оплачен
.status-processing { color: #f59e0b; } // В обработке
.status-canceled { color: #ef4444; }  // Отменен
.status-new { color: #6b7280; }         // Новый заказ
```

## Примеры использования

### Успешный поиск
```
Ввод: "314c92fb-000f-5000-b000-1c4140d12338"
Отображение:
- Карточка заказа с деталями
- Статус: "Оплачен" (зеленый)
- Детали: товар, цена, дата, способ доставки
- Информация о клиенте
```

### Заказ не найден
```
Ввод: "invalid_payment_id"
Отображение:
- Карточка ошибки
- Сообщение: "Заказ с таким ID не найден. Проверьте введенный ID или свяжитесь с поддержкой."
- Кнопка "Попробовать снова"
```

## Развертывание

## Локализация
Все текстовые сообщения на русском языке.

## Безопасность
- Валидация на пустоту перед отправкой запроса в Firebase
- Отсутствие прямого доступа к базе данных (через Firebase SDK)
